# Generative Agents and Memory-Based Retrieval

This repository contains assignment work for building and testing memory retrieval for generative agents, then extending it to a personal life-log Q&A setting.

## Project Structure

- `section1.ipynb`: Build text embeddings from memory nodes (`nodes.json`) and write `embeddings.json`.
- `section2.ipynb`: Baseline memory retrieval using recency, importance, relevance, and type weighting.
- `section3.ipynb`: Retrieval strategy experiments with different weight configurations per query.
- `section4.ipynb`: Life-log Q&A pipeline on Klaus data; retrieves memories and prints grounded prompts for an LLM.
- `section5.ipynb`: Personal life-log agent on synthetic Alex Chen data (`personal.json`) and `personal_embeddings.json`.
- `nodes.json`: Original generative-agents memory stream (593 nodes).
- `embeddings.json`: Precomputed embeddings for unique memory keys in `nodes.json` (202 vectors, 384-dim).
- `personal.json`: Custom personal life-log dataset (60 nodes).
- `personal_embeddings.json`: Precomputed embeddings for `personal.json` (60 vectors, 384-dim).
- `Srinath_Muppala_ASG2.md`: Full assignment report with outputs and analysis.
- `main.py`: Placeholder entrypoint generated by project scaffolding.

- Unique embedding keys: `60`

Each node includes fields such as `type`, `created`, `description`, `embedding_key`, and `poignancy`.

## Retrieval Scoring (Sections 2-5)

Memories are ranked with a weighted score:

`score = type_weight * (W_RECENCY * recency + W_IMPORTANCE * importance + W_RELEVANCE * relevance)`

- `recency`: exponential decay using half-life (`HALF_LIFE_HOURS`, default 24h in Section 2/3).
- `importance`: normalized `poignancy` (1-10 -> 0-1 scale).
- `relevance`: cosine similarity between query embedding and memory embedding.
- `type_weight`: favors memory types (Section 2/3 uses `thought: 2.0`, `chat: 1.7`, `event: 1.0`).
- `REL_MIN`: minimum similarity filter (`0.62` in Sections 2/3) to reduce low-signal memories.

## Setup

This project uses Python 3.12 and `uv`.

```bash
uv sync
```

Dependencies in `pyproject.toml`:

- `sentence-transformers`
- `ipykernel`

To run notebooks, open this folder in VS Code/Jupyter and execute cells in order.

## Recommended Execution Order

1. Run `section1.ipynb` to (re)generate `embeddings.json` from `nodes.json`.
2. Run `section2.ipynb` for baseline retrieval and initial query tests.
3. Run `section3.ipynb` for weight tuning experiments.
4. Run `section4.ipynb` to generate grounded prompts and answer Klaus life-log questions.
5. Run `section5.ipynb` to build/test the personal life-log agent with `personal.json`.

## Notes on LLM Q&A (Sections 4 and 5)

Sections 4 and 5 retrieve top memories and print a structured prompt template intended for copy-paste into an LLM (for example ChatGPT/Gemini). The notebooks enforce grounded answers by requiring evidence IDs from retrieved memories.
